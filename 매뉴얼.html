1. 사전 준비 — Docker Desktop 설치

시스템 실행을 위해 Docker Desktop 설치가 필요합니다. (최초 1회)

1
Docker Desktop 다운로드 https://www.docker.com/products/docker-desktop/ 접속 후 Windows 버전 다운로드
2
설치 실행 다운로드된 설치 파일을 실행하고 안내에 따라 설치 진행
3
PC 재시작 설치 완료 후 반드시 PC를 재시작하세요
4
Docker Desktop 실행 확인 재시작 후 Docker Desktop을 실행하여 고래 아이콘이 트레이에 표시되면 준비 완료
2. 최초 설치 및 실행
2-1. 소스 압축 해제

제공받은 압축 파일을 원하는 위치에 압축 해제합니다.

📁 압축 해제 후 폴더 구조
ALM-SYSTEM-250915/
  ├── 실행.bat  ← 시스템 실행 파일
  ├── backend/     ← 서버 소스
  ├── frontend/    ← 화면 소스
  └── docker-compose.yml
2-2. 아이디 / 비밀번호 설정

backend 폴더 안의 .env 파일을 메모장으로 열어 아래 두 줄을 수정합니다.

# 로그인 정보 — 원하는 아이디와 비밀번호로 변경
LOGIN_ID=변경할아이디
LOGIN_PASSWORD=변경할비밀번호
⚠ 주의
.env 파일은 숨김 파일일 수 있습니다. 탐색기에서 "숨김 항목 표시"를 체크하거나, 폴더 주소창에 직접 경로를 입력하여 접근하세요.
2-3. 시스템 실행

압축 해제한 폴더에서 실행.bat 파일을 더블클릭합니다.

1
실행.bat 더블클릭 "시스템을 시작하는 중입니다" 로딩 창이 표시됩니다
2
최초 실행 — 자동 설치 대기 (5~10분) 처음 실행 시 필요한 파일을 자동으로 다운로드합니다. 인터넷 연결이 필요합니다.
3
브라우저 자동 실행 설치가 완료되면 브라우저가 자동으로 열리며 로그인 화면이 표시됩니다
✅ 두 번째 실행부터는 1~2분 이내에 시작됩니다
최초 1회만 긴 시간이 소요되며, 이후에는 빠르게 실행됩니다.

5. 시스템 종료

시스템을 종료할 때는 반드시 아래 방법을 사용하세요.

1
Windows 트레이 아이콘 확인 화면 우측 하단 트레이 영역(시계 왼쪽)에서 시스템 아이콘을 찾습니다. 숨겨진 경우 ∧ 버튼을 클릭하세요.
2
아이콘 우클릭 → "종료" 선택 종료를 클릭하면 Docker 컨테이너가 자동으로 정지됩니다
ℹ 트레이 아이콘 기능
· 더블클릭 : 브라우저에서 웹 페이지 다시 열기
· 우클릭 → 웹 열기 : 브라우저에서 웹 페이지 열기
· 우클릭 → 종료 : 시스템 종료
6. 데이터 보관

입력한 모든 데이터(사원 정보, 연차 내역, 로그 등)는 PC 로컬에 안전하게 보관됩니다.

상황데이터 보존 여부
시스템 종료 (트레이 아이콘 → 종료)✅ 보존
PC 재시작 후 다시 실행✅ 보존
Docker Desktop 재시작✅ 보존
소스 폴더 삭제✅ 보존 (Docker 볼륨에 별도 저장)
docker compose down -v 명령 실행❌ 삭제됨
🚫 절대 실행 금지
docker compose down -v 명령은 모든 데이터가 삭제되므로 절대 실행하지 마세요.
7. 주의사항
항목내용
Docker Desktop 실행 순서반드시 Docker Desktop이 먼저 실행된 상태에서 실행.bat을 클릭하세요. (또는 실행.bat이 자동으로 Docker를 시작합니다)
인터넷 연결최초 실행 시 인터넷 연결이 필요합니다. 이후에는 오프라인으로 사용 가능합니다.
접속 주소http://localhost:8080 으로만 접속 가능합니다.
동시 접속단일 사용자 전용 시스템입니다. 여러 기기에서 동시 접속은 지원하지 않습니다.
로그인 유지 시간3시간 이후 자동 로그아웃됩니다. 재로그인 후 계속 사용하세요.
포트 충돌8080 포트를 사용 중인 다른 프로그램이 있으면 실행이 안 될 수 있습니다.
